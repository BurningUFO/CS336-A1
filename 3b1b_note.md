# 深度学习之神经网络的结构

网络为第一层（输入层）和第二层之间的**每一条连接**分配了一个数字，称为**权重 (Weight)**。

- 如果是正权重（视频中的绿色像素），代表“我希望这里有光亮”。
- 如果是负权重（视频中的红色像素），代表“我希望这里是暗的”。
- 计算方式：将第一层所有 784 个神经元的激活值，分别乘以它们对应的权重，然后全部加起来，得到一个**加权和 (Weighted Sum)**。

```
double weighted_sum = 0.0;
for(int i = 0; i < 784; i++) {
    weighted_sum += input_pixels[i] * weights[i];
}
```

这个 `weights` 数组本质上就是一个**滤波器 (Filter)或者叫掩码 (Mask)**。当输入的图像像素点刚好和正权重的位置重合时，乘积就很大；和负权重重合时，就会被抵消。这就是深度学习中“检测特征”的底层数学原理。

上面算出的 `weighted_sum` 可能是一个非常大或非常小的任意实数（比如 152.3 或 -89.1）。但我们之前规定过，神经元的激活值必须在 $0$ 到 $1$ 之间。因此，我们需要一个数学函数把任意实数“压缩”到 $(0, 1)$ 区间。 **Sigmoid 函数**（也叫逻辑曲线）：输入越负，输出越接近 0；输入越正，输出越接近 1。

有时候，仅仅是加权和大于 0 并不足以让神经元兴奋。我们可能希望这个匹配得分非常高（比如大于 10）时，神经元才被点亮。这时候我们就引入了**偏置 (Bias)**。我们在算完加权和之后，加上一个负数（比如 -10）作为偏置，然后再送进 Sigmoid 函数。

**案例/辅助理解**：偏置相当于一个“触发阈值 (Threshold)”。

用 `if` 条件语句来类比：`if (weighted_sum + bias > 0)` 神经元才开始产生有意义的激活。

*结合起来看参数规模*：仅仅是第一层到第二层（784 个输入神经元，16 个隐藏层神经元），就需要 $784 \times 16 = 12544$ 个权重，加上 16 个偏置，仅这一步就有过万个“旋钮”！整个简单网络有约 13000 个参数。所谓的“机器学习”，就是计算机在不断微调这 13000 个变量，直到它们能完美协作。

如果用数学符号把每一条连接的公式都写出来，会极其繁琐。数学家和计算机科学家使用**线性代数**来降维打击：

把一整层的激活值打包成一个向量 $\mathbf{a}^{(0)}$；

把所有的权重打包成一个二维矩阵 $\mathbf{W}$；

把所有的偏置打包成一个向量 $\mathbf{b}$。

那么下一层的状态就可以用一个极度简洁的公式表达：**$\mathbf{a}^{(1)} = \sigma(\mathbf{W} \mathbf{a}^{(0)} + \mathbf{b})$**

这是极其硬核且关键的一步！在 C++ 层面，如果你写双重 `for` 循环去计算那一万多个连接，CPU 跑起来会很慢。但是当你把它抽象成**矩阵乘法 (Matrix Multiplication)** 时，由于矩阵运算的内存连续性和高度并行性，底层的运算库（比如 CUDA 或 BLAS）可以调用 GPU 的成千上万个流处理器同时进行计算。这就是为什么深度学习必须依赖线性代数，以及为什么我们需要显卡来跑 AI。

前面讲的 Sigmoid 函数（把值压缩到 0 到 1 之间）其实已经是“老古董”了。在现代神经网络（包括你要学的大语言模型底层的 Transformer）中，隐藏层极少使用 Sigmoid，而是广泛采用了一种极其简单的激活函数：**ReLU (Rectified Linear Unit，线性整流函数)**。

- **公式**：$a = \max(0, z)$ （其中 $z$ 是加权和加偏置的结果）。
- **含义**：如果输入信号是正数，就原封不动地输出；如果输入信号是负数，就直接归零（截断）。

**案例/辅助理解**：为什么现代 AI 抛弃了看起来很高级的 Sigmoid 曲线，选择了这么“粗暴”的 ReLU？

从 C/C++ 底层性能的角度看，计算一次 Sigmoid 需要调用 `1.0 / (1.0 + exp(-x))`，涉及复杂的浮点指数运算，非常消耗算力。而 ReLU 只需要极其简单的一行逻辑：

C

```
double relu(double x) {
    return (x > 0) ? x : 0.0;
}
```

它不仅在 CPU/GPU 上的计算速度快得惊人，而且在训练深层网络时，能有效解决一种叫做“梯度消失”的致命问题（这部分会在后续训练原理中展开）。大模型动辄上百亿、千亿参数，这种底层计算效率的提升是决定性的。



# 深度学习之梯度下降法

所谓“学习”，绝不是写几条 `if-else` 规则教计算机认字，而是设计一种**算法**，让这 13000 个参数能够根据大量已知的、带标签的数据（视频中提到了经典的 MNIST 手写数字数据集），**自动进行自我修正**。

要让计算机自我修正，首先必须量化它当前“错得有多离谱”。视频引出了深度学习中最核心的概念之一：**代价函数（也叫损失函数 Loss Function）**。

- 针对**单张图片**：我们把网络输出的 10 个预测值，与我们期望的 10 个理想值（除了正确数字是 1，其他全是 0）进行对比。计算它们**差值的平方和**。这个和越小，说明网络越自信且越准确；越大，说明网络完全在胡猜。
- 针对**整个网络**：把这上万张训练图片全部跑一遍，算出每一张的代价，然后求**平均值**。这个平均代价，就是评价这 13000 个参数当前设置得好坏的唯一标准。

在 C++ 或 Python 中，这其实就是一个遍历评估的过程：

Python

```python
def calculate_cost(network, training_data):
    total_cost = 0.0
    for image, label in training_data:
        predictions = network.forward(image)
        # 计算预测值与真实标签的差异（差值的平方和）
        cost = sum((pred - expected)**2 for pred, expected in zip(predictions, label))
        total_cost += cost
    return total_cost / len(training_data) # 返回平均代价	
```

这个 `calculate_cost` 返回的数字如果巨大，就是在告诉计算机："Bad computer!"。

我们在第一期视频中“一厢情愿”地设想：第一层学小边缘，第二层学形状，第三层拼凑数字。但当你真正把训练好的网络权重可视化时，你会发现它们看起来完全是**毫无规律的随机噪点** ，仅仅在中间隐约有一点点轮廓。

**残酷的现实测试**：如果你给这个训练好的网络输入一张完全由黑白雪花点组成的“纯噪音”图片，它不仅不会表示“我不知道这是啥”，反而会极其自信地给出错误答案（比如判定它 100% 是数字 5）。

为什么会这样？因为网络并没有像人类那样真正建立“什么是数字 5”的逻辑概念。它只是在那个拥有 13000 个变量的高维数学空间里，瞎猫碰死耗子般地找到了一个能把误差降得很低的**局部最优解 (Local Minimum)**。它发现了一种**统计学上的捷径**。这就好比你在准备期末考试时，没有真正弄懂操作系统底层的内存分配原理，而是死记硬背了历届真题的答案规律，碰上原题能拿高分，碰上没见过的变形题直接翻车。

现代的深度神经网络拥有极其庞大的参数量，庞大到它甚至有能力**死记硬背**下几万张毫无规律的垃圾图片（哪怕你把标签全打乱）。

但是，**真实的、有结构的数据**（比如真正的手写笔迹）会在高维空间中形成一个非常有利于梯度下降的“地形” 。在真实数据构成的代价函数表面，梯度下降算法能非常迅速且顺滑地“滚入”一个高质量的最低谷；而如果是随机打乱的假数据，寻找谷底的过程就会像在平地上找坑一样极其艰难。

# 深度学习之反向传播算法

我们知道要寻找代价函数的“负梯度”来下坡。但 13000 维的梯度向量到底怎么算？**反向传播就是用来计算这个巨型梯度的核心算法**。

视频强调了一个极其重要的直觉：不要去想 13000 维的空间方向，而要把梯度的每一个分量看作是代价函数对某个特定权重/偏置的**敏感度**。

假设你算出权重 $W_1$ 对应的梯度值是 $3.2$，而权重 $W_2$ 对应的是 $0.1$。这意味着，如果你把 $W_1$ 稍微拨动一点点，代价函数产生的变化（下降幅度）将是拨动 $W_2$ 的 32 倍！在底层 C/C++ 代码优化中，这就好比我们找到了系统的“性能瓶颈”，把算力投资在 $W_1$ 的更新上“性价比”最高（Most bang for your buck）。

如果我们要让 $a_{out}$ **变大**，数学上只有三条路可以走，这也就是该神经元发出的三个“请求”：

1. **增加偏置 (Bias)**：这是最直接的。
2. **增加权重 (Weights)**：重点来了！应该优先增加那些与**上一层高激活值神经元**相连的权重。因为 $(大权重 \times 大激活值)$ 才能最大化地提升总和。这印证了赫布理论 (Hebbian Theory)：“一起激发的神经元连在一起 (Neurons that fire together, wire together)”。
3. **改变上一层的激活值 (Activations)**：如果前一层的某个神经元连着一个正权重，那就要求它变亮；如果连着负权重，就要求它变暗。

前一段我们只看了一个神经元的诉求。但输出层有 10 个神经元！为了降低总误差，代表“8”的神经元可能希望上一层的节点 A 变暗，而代表“2”的神经元希望节点 A 变亮。网络该听谁的？ 答案是：**按权重和迫切程度进行“加权投票”**。网络会把这 10 个输出神经元对节点 A 的“修改诉求”全部加起来。那些当前误差最大（最迫切）且连接权重最高（声音最大）的神经元，它的投票权就越重。

**核心动作（递归思想）**：一旦我们通过这种投票机制，算出了**倒数第二层所有神经元**究竟应该变亮还是变暗（以及改变的幅度），我们就可以把倒数第二层当成新的“输出层”，再次应用上一段学到的“调节三大旋钮”的方法，去求出倒数第三层的变化……  这种把误差期望从后往前一层一层倒推的过程，就是大名鼎鼎的**反向传播 (Backpropagation)**！

如果你只拿一张“2”的图片做反向传播，并更新了那 13000 个参数，网络的确会在识别这张“2”时表现更好，但代价是它看什么都像“2”。 为了让它真正学会识别所有数字，我们必须把上述极其繁琐的反向传播过程，在所有的几万张训练图片上都跑一遍。记录下每一张图片对这 13000 个参数提出的“微调建议”，最后把所有的建议**求平均值**。

为了在参数空间里仅仅“走一步”，竟然要让 CPU/GPU 把几万张图片的正向计算和反向传播全部算一遍？这太慢了！ 深度学习祭出了一个伟大的妥协方案：**Mini-batches (小批量)**。 系统会把几万张图片随机打乱，然后切分成很多个小包（比如每 100 张图片一个 batch）。网络只看这 100 张图片，就算出一个近似的梯度，然后立刻更新参数（走一步）。

# 反向传播的微积分原理

我们定义几个核心变量（右上角的 $(L)$ 代表当前层，$(L-1)$ 代表上一层）：

1. **输入**：上一层的激活值 $a^{(L-1)}$
2. **参数**：这一层的权重 $w^{(L)}$ 和偏置 $b^{(L)}$
3. **线性加权和**：$z^{(L)} = w^{(L)}a^{(L-1)} + b^{(L)}$
4. **激活输出**：$a^{(L)} = \sigma(z^{(L)})$ （经过 Sigmoid 或 ReLU 等处理）
5. **单样本代价**：$C_0 = (a^{(L)} - y)^2$ （其中 $y$ 是真实标签）

![Snipaste_2026-03-14_10-48-46](Snipaste_2026-03-14_10-48-46.png)

我们的终极目标是求出：**代价函数 $C_0$ 对权重 $w^{(L)}$ 的敏感度（也就是偏导数 $\frac{\partial C_0}{\partial w^{(L)}}$）**。 由于 $w^{(L)}$ 并不能直接算出 $C_0$，它们中间隔着 $z$ 和 $a$。所以必须使用微积分的链式法则，将这个大偏导拆解为三个微小推导的乘积：

 $\frac{\partial C_0}{\partial w^{(L)}} = \frac{\partial z^{(L)}}{\partial w^{(L)}} \times \frac{\partial a^{(L)}}{\partial z^{(L)}} \times \frac{\partial C_0}{\partial a^{(L)}}$

现在我们要把上面的三个“传动比”具体算出来：

1. $\frac{\partial C_0}{\partial a^{(L)}} = 2(a^{(L)} - y)$ ：这代表了网络的输出离正确答案有多远。错得越离谱，这个值越大，传导的误差就越猛。
2. $\frac{\partial a^{(L)}}{\partial z^{(L)}} = \sigma'(z^{(L)})$ ：这是激活函数的导数。如果加权和处于激活函数的平缓地带，这个导数趋近于 0，会导致“梯度消失”。
3. $\frac{\partial z^{(L)}}{\partial w^{(L)}} = a^{(L-1)}$ ：这是最精彩的一步！加权和 $z$ 对权重 $w$ 求导，剩下的常数项竟然刚好是**上一层的激活值**！这在数学上完美印证了上一集提到的“一起激发的神经元连在一起”——上一层的神经元越亮（$a^{(L-1)}$越大），当前权重的微调对最终结果的影响就越剧烈。

**偏置的导数与向后传递 (Bias and Backward Propagation)**

- **求偏置的导数**：公式完全一样，只需把最后一步换成 $\frac{\partial z^{(L)}}{\partial b^{(L)}}$。因为 $z = wa + b$，$z$ 对 $b$ 求导结果恒为 **$1$**。
- **为上一层铺路**：为了让反向传播能继续往 $(L-1)$ 层倒推，我们还需要求出代价函数对**上一层激活值 $a^{(L-1)}$** 的偏导。依然是链式法则，只是把最后一步换成了 $\frac{\partial z^{(L)}}{\partial a^{(L-1)}} = w^{(L)}$。拿到这个值，网络就可以继续向更浅层传递误差了。

我们终于要面对真实的、每一层有多个神经元的网络了。数学原理其实**没有任何改变**，唯一的难点在于我们需要引入更多的下标（Indices）来精确追踪“哪条边连着哪个节点”。

- 视频约定：用 $k$ 代表上一层 $(L-1)$ 的第 $k$ 个神经元；用 $j$ 代表当前层 $(L)$ 的第 $j$ 个神经元。
- **权重的特殊命名法**：连接 $k$ 和 $j$ 的权重被记作 $w_{jk}^{(L)}$。注意看，下标是反过来的（先目标 $j$，后来源 $k$）！为什么？因为这样恰好能完美对应线性代数中的矩阵乘法 $\mathbf{W}\mathbf{a}$，每一行代表当前层一个神经元的所有输入权重。
- **代价函数升级**：因为输出层现在有多个神经元，所以总代价变成了所有输出神经元误差的累加和：$C_0 = \sum_{j} (a_j^{(L)} - y_j)^2$。

尽管网络变复杂了，但如果我们只盯着一条特定的边（也就是某一个特定的权重 $w_{jk}^{(L)}$），求它对代价函数影响的链式法则公式，和刚才的单节点网络**完全一样**！

$$\frac{\partial C_0}{\partial w_{jk}^{(L)}} = \frac{\partial z_j^{(L)}}{\partial w_{jk}^{(L)}} \frac{\partial a_j^{(L)}}{\partial z_j^{(L)}} \frac{\partial C_0}{\partial a_j^{(L)}}$$

整个反向传播微积分最精妙、最容易卡壳的地方就在这里！当我们想求**上一层某个神经元 $a_k^{(L-1)}$ 的导数**（即它对最终误差要负多少责任）时，情况变了。

因为神经元 $k$ 发出的信号，通过不同的权重，同时连接到了下一层**所有**的 $j$ 个神经元上。它对最终误差的影响是“多管齐下”的。

根据多元微积分的法则，如果一个变量通过多条路径影响最终结果，你必须把**所有路径的导数加起来**：

$$\frac{\partial C_0}{\partial a_k^{(L-1)}} = \sum_{j} \left( \frac{\partial z_j^{(L)}}{\partial a_k^{(L-1)}} \frac{\partial a_j^{(L)}}{\partial z_j^{(L)}} \frac{\partial C_0}{\partial a_j^{(L)}} \right)$$

# 直观解释Transformer

大模型（如 GPT-3）最底层的机制其实非常单一：**预测下一个词 (Predict the next word)**。它输出的不是一个确定的词，而是一个**概率分布 (Probability Distribution)**。

**Transformer 数据流的 5 步宏观架构** 这是 CS336 中必须要在大脑里形成肌肉记忆的数据流水线！视频极其清晰地展示了数据是如何穿过 Transformer 的 ：

- **Step 1: 分词 (Tokenization)**
	- **概念**：计算机不认识完整的英文句子。输入的第一步是把句子切碎成 **Tokens (标记)**。一个 Token 可以是一个完整的词，也可以是半个词（词根、词缀），甚至是标点符号。
- **Step 2: 向量化 (Embedding)**
	- **概念**：查表操作。将每一个 Token 转换成一个超长的一维数组（**高维向量 Vector**）。这个数组里的数字，在数学空间中编码了这个词的“初始含义”。
- **Step 3: 注意力层 (Attention Block) —— 词与词的“交流”**
	- **概念**：这是 Transformer 的灵魂！初始的向量只包含字面意思，但语言是有上下文的（比如 machine learning model 和 fashion model 中的 model 含义完全不同）。
	- **底层逻辑**：Attention 机制允许这组向量在空间中互相“通信”。`model` 这个向量会去查看前面的 `machine` 和 `learning` 向量，然后吸收它们的信息，**更新并改变自己的向量值**。
- **Step 4: 多层感知机层 (Multi-Layer Perceptron, MLP) / 前馈层 (Feed Forward)**
	- **概念**：Attention 层让词语之间交换了信息，而 MLP 层则是让每个词的向量**独立地**进行自我消化和升华。
	- **底层逻辑**：在这一步，各个向量**不再互相交流**。它们被并行地送入一个相同的全连接神经网络。你可以把它理解为程序中的一个 `map()` 函数：对数组中的每一个元素独立执行一套复杂的数学映射。
- **Step 5: 重复交替与最终预测 (Repetition and Prediction)**
	- **概念**：数据会在 Attention 块和 MLP 块之间交替穿梭几十次（多层结构）。在网络的绝对末端，序列中**最后一个词**的向量，已经吸饱了前面所有词的上下文灵魂。最后对这最后一个向量进行一次矩阵乘法，映射到词表大小，得出下一个词的概率分布。

文字究竟是怎么变成数字的？靠的是**词嵌入矩阵 (Embedding Matrix, $W_E$)** 。 这是一个巨大无比的二维数组，大小是 `12288 行 × 50257 列`（GPT-3）。它的**每一列**，就对应词表中一个确定的词。当输入句子中的一个词时，计算机只是做了一个极度简单的**查表操作 (Lookup Table)**，把对应的那个长度为 12288 的一维数组提取出来。 *仅仅这一个矩阵，参数量就达到了 $50257 \times 12288 \approx 6.17$ 亿个！* * **高维语义空间 (Semantic Space)**：这 12288 个数字并不是随机的，它们在高维空间中定义了一个确切的坐标 。在经历了漫长的预训练后，**空间中的“方向”就代表了“语义”**。

既然词变成了高维空间里的向量，我们怎么在数学上判断两个词的含义“接不接近”？ 这就用到了线性代数里的**点积 (Dot Product)**（也就是把两个数组对应位置的元素相乘再求和）。

- 点积为 **正且很大**：两个向量方向相似，语义高度相关。
- 点积为 **零**：向量相互垂直（正交），语义毫不相干。
- 点积为 **负**：向量方向相反。

前面我们说 `bank` 查表得到的是一个“出厂默认”的词向量。在这个阶段，它是不带感情色彩的。但是，当它穿过 Transformer 的网络（Attention 块）时，它会疯狂地“吸收”周围的词（比如 `river` 或 `money`）的信息，并在高维空间中改变自己的方向。到了网络的最后一层，这个向量已经变成了一个极其复杂的“复合体”。

**工程限制 (Context Size)**：网络一次性能处理的 Token 数量是固定的，这被称为“上下文大小”。GPT-3 的限制是 **2048**。

**逆向还原：反嵌入矩阵 (Unembedding Matrix) 与 Logits**

- **概念深度解析**：当数据流到达最后一层时，我们要用序列中的**最后一个向量**（它已经吸饱了整个句子的上下文灵魂）来预测下一个词。

	怎么把这个 12288 维的向量变回人类语言？我们需要另一个极其庞大的矩阵：**反嵌入矩阵 (Unembedding Matrix, $W_U$)**。

	它的维度是 `50257 行 × 12288 列`（又是 6.17 亿个参数！）。用这个矩阵去乘以最后一个 12288 维的向量，我们会得到一个长度为 50257 的一维数组。这个数组里的每一个数字，代表了词表中对应词汇的“得分”。

- **专业术语 (Logits)**：在这个阶段，算出来的这些得分有正有负，且不加总为 1，在深度学习中，这些未经处理的原始得分被称为 **Logits**。

Logits 是毫无约束的实数，但我们需要的是一个 **概率分布 (Probability Distribution)**（所有的值必须在 **0** 到 **1** 之间，且总和为 **1**）。 这里引出了 AI 领域最著名的激活函数之一：**Softmax**。它的数学公式极其优雅，它将自然常数 $e$ 作为底数，对所有的 Logits 进行指数化，然后归一化。

**温度系数 (Temperature, $T$)**：

这是大模型 API（比如 OpenAI API）中最常用的一个调节参数。它被放在了 Softmax 指数部分的**分母**上：

$$P(i) = \frac{e^{x_i / T}}{\sum_{j} e^{x_j / T}}$$

- **当 $T = 1$ 时**：标准的 Softmax。
- **当 $T \to 0$ 时（低温）**：这被称为贪婪搜索 (Greedy Search)。得分最高的那一个词会占据近乎 **100%** 的概率，其他词全部趋近于 0。模型的回答会极其确定、死板。
- **当 $T > 1$ 时（高温）**：分布曲线被“压平”了。原本得分较低的词，现在也有了被选中的概率。模型变得更有“创造力”，甚至开始胡言乱语（幻觉 Hallucination）。

# 注意力机制

在上一集我们知道，刚经过 Embedding 查表的词向量是不带上下文的。比如 `mole` 这个词（可以指动物鼹鼠、化学摩尔、皮肤黑痣、或者间谍），在进入网络的第一步时，它们的 12288 维初始向量是**完全一模一样**的。 Attention 的终极目标，就是根据周围的词（比如 `American spy mole` 或是 `biopsy of mole`），算出需要给这个初始向量**加上一个怎样的“增量”**，把它往特定的高维语义方向（比如间谍方向，或者医学方向）推过去。

视频用了一句话作为贯穿始末的例子："a fluffy blue creature roamed..."（一只毛茸茸的蓝色生物漫游...）。 在这里，名词 `creature` 迫切需要吸收形容词 `fluffy` 和 `blue` 的语义，才能让自己变成一个“毛茸茸蓝色生物”的复合向量。 但问题是，计算机怎么知道 `creature` 应该去向 `fluffy` 索要信息，而不是去向 `roamed` 索要信息呢？这就需要一套完美的“提问与回答”机制。

为了让词与词互相寻找，网络为每一个词生成了两个全新的、维度较小（比如 128 维）的向量：

1. **Query (查询向量, Q)**：代表这个词在“寻找什么”。相当于它大喊：“我是个名词，谁是修饰我的形容词？”

	- *计算方式*：将原始的嵌入向量乘以一个专门的权重矩阵 $W_Q$。

2. **Key (键向量, K)**：代表这个词的“身份标签”。相当于 `fluffy` 在举手：“我是一个前置形容词！”

	- *计算方式*：将原始的嵌入向量乘以另一个专门的权重矩阵 $W_K$。

		（注：矩阵 $W_Q$ 和 $W_K$ 就是大模型在训练时疯狂学习和微调的参数）。

如何判断一个词的 Query 和另一个词的 Key 是否匹配？回到我们在上一集学到的线性代数杀手锏：**点积 (Dot Product)**。 将 `creature` 的 Query 向量与句子中所有其他词的 Key 向量进行点积运算：

- 如果点积结果是一个**巨大的正数**：说明高度匹配（比如 `creature` 和 `fluffy`），`creature` 需要高度关注（Attend to）这个词。
- 如果点积结果是零或负数：说明不匹配（比如 `creature` 和 `the`），直接无视。

在上一段中，我们通过 Query 和 Key 的点积得到了一张充满任意实数（从 $-\infty$ 到 $+\infty$）的得分网格。为了把这些得分变成可以用来做“加权平均”的权重，我们需要对网格的**每一列**应用 **Softmax** 函数。 经过 Softmax 处理后，原本绝对值很大的正数会无限逼近 1，负数会无限逼近 0，且每一列的得分总和严丝合缝地等于 1。这就形成了一个标准的**概率分布**。 *(注：视频提到一个小细节，为了数值稳定性，在做 Softmax 之前会把点积除以 $\sqrt{d_k}$，也就是 Key 维度的平方根，这是标准 Transformer 公式的一部分：$\text{Softmax}(\frac{QK^T}{\sqrt{d_k}})$)*。

**掩码机制 (Masking)**

- **概念深度解析**：

	在训练大模型时，为了极致的效率，我们是一次性把整句话喂给 GPU，让它**同时**预测每一个位置的下一个词。但是，当模型在预测 `creature` 后面的词时，它**绝对不能**提前看到句子后面的 `roamed` 等词，否则这就叫“数据泄露”或“作弊”。

- **底层实现思路**：我们如何在矩阵运算中强行屏蔽掉未来的词？

	答案是**掩码 (Masking)**。 在应用 Softmax 之前，我们强行把矩阵中代表“未来词影响当前词”的那些位置（也就是矩阵的右上三角半部分）的值，全部手动设置为 $-\infty$。这样一来，经过 Softmax 计算后，这些位置的权重就会彻底变成 **0**。

	在 C/C++ 层面，这其实就是用一个双重 `for` 循环，当遍历到 `if (j > i)`（即后面的词 `j` 试图影响前面的词 `i`）时，直接赋值 `score = -INFINITY;`。

**值向量 (Value) 与特征的物理转移**

- **概念深度解析**：我们现在有了权重矩阵，知道 `creature` 有 90% 的注意力放在了 `fluffy` 上。但是，**到底是什么数据被加到了 `creature` 身上？**

	这里引出了第三个、也是最后一个核心矩阵：**值向量矩阵 (Value Matrix, $W_V$)**。

	每个词除了算出 Query 和 Key，还会通过乘以 $W_V$ 算出一个 **Value (值) 向量**。

	- **Q (Query)**：提出需求（“谁是修饰我的形容词？”）
	- **K (Key)**：表明身份（“我是形容词！”）
	- **V (Value)**：**实际传递的有效载荷 (Payload)**（包含“毛茸茸”这个概念的实际高维数据）。

- **物理合并**：`creature` 的新向量，就是周围所有词的 Value 向量的**加权求和**（权重就是刚才算出的 Attention 概率）。算出这个增量 $\Delta E$ 后，把它直接加到 `creature` 原本的词嵌入向量上：$E_{new} = E_{old} + \Delta E$。至此，`creature` 终于完成了进化！

**单头注意力的参数矩阵拆解与降维**

- **概念深度解析**：在刚才的推导中，我们完成了一次完整的**单头注意力 (Single-Head Attention)** 操作。来盘点一下参数：

	$W_Q$ 和 $W_K$ 将 12288 维的输入降维成了 128 维，参数量不大。但是 $W_V$ 呢？如果它直接在 12288 维的空间里做映射，参数量会爆炸（$12288 \times 12288$）。

	为了优化性能，工程上巧妙地把 $W_V$ 拆成了两个小矩阵相乘（低秩近似）：

	1. **Value Down**：先从 12288 维降到 128 维（在这个小空间里完成刚才所有的加权求和操作）。

	2. **Value Up**：再把求和后的结果从 128 维映射回 12288 维。

		*(注：在大多数经典论文和框架中，这个 Value Up 矩阵通常被称为输出矩阵 $W_O$)*。这四个矩阵（Q, K, V_down, V_up）加起来大约有 630 万个参数。

**为什么需要“多头”？（捕捉多维度的语境）**

- **概念深度解析**：前面我们推导的“单头注意力”，只能让模型学会**一种**词汇之间的关联（比如“形容词修饰名词”）。但人类语言极其复杂，同样是 `Harry` 这个词，如果上下文中出现了 `wizard` (巫师)，它应该指向《哈利·波特》；如果上下文中出现了 `Queen` 和 `William`，它应该指向英国的哈里王子。

	单一的 $W_Q$、$W_K$、$W_V$ 矩阵容量有限，无法同时兼顾语法关联、人物代词指代、甚至情感色彩等无数种维度的语义关系。

**多头注意力机制 (Multi-Head Attention)**

- **概念深度解析**：既然一个头不够用，Transformer 的暴力美学就体现出来了——**直接开多个头并行计算！**

	在 GPT-3 中，每一个 Attention 模块内部，包含了整整 **96 个**独立并行的注意力头 (Attention Heads)。这意味着，网络里有 96 套完全不同的 $W_Q$、$W_K$ 和 $W_V$ 矩阵，各自算出一份截然不同的注意力网格（Attention Pattern）。

- **底层 C++ 并发思维**：

	用你的专业语言来解释，这简直就是一个完美的 GPU 并行计算模型！

	C++

	```
	// 假设我们启动 96 个线程同时处理
	std::vector<std::thread> heads;
	std::vector<Vector> head_updates(96);
	
	for (int i = 0; i < 96; i++) {
	    heads.emplace_back([i, &head_updates]() {
	        // 每个线程拥有自己独立的 W_Q, W_K, W_V
	        // 计算当前头的 Q*K 点积，过 Softmax，最后乘以 V 得到建议的增量
	        head_updates[i] = compute_single_head_attention(); 
	    });
	}
	// 等待所有线程完成
	join_all(heads);
	
	// 把 96 个头提取到的 96 份不同维度的语义增量，全部加到原向量上！
	for (int i = 0; i < 96; i++) {
	    creature_embedding += head_updates[i]; 
	}
	```

	通过这种机制，`creature` 这个词瞬间吸收了 96 种不同层面的上下文理解。

**工程实现的真相 —— 输出矩阵 $W_O$ (Output Matrix)**

- **概念深度解析**：3Blue1Brown 在这里特别澄清了一个读经典论文《Attention is all you need》时容易踩坑的细节。

	在教学中，他把 $W_V$ 拆成了 Value Down 和 Value Up。但在真实的深度学习框架（如 PyTorch）底层，为了极致的矩阵乘法优化，我们会把这 96 个头算出来的小维度结果**拼接 (Concatenate)** 在一起，然后统一乘以一个巨大的**输出矩阵 (Output Matrix, $W_O$)**。

	这不仅减少了细碎的矩阵乘法次数，也让内存访问（Memory Access）在显存中更加连续。

# How might LLMs store facts

**模型的“记忆黑盒” (MLP 块的定位)**

- **概念深度解析**：之前我们知道 Attention 让词语互相交流。但当输入 `Michael Jordan plays the sport of...` 时，模型预测出 `basketball` 依靠的不是句子本身的语法语境，而是它死记硬背的客观事实。Google DeepMind 的研究指出，这些百科全书般的事实，大概率就储存在 **多层感知机 (MLP)** 中。
- **运行机制**：在经历过一轮 Attention 之后，每一个词向量都会**独立地**（不再与其他词交流）进入 MLP 块进行自我处理。处理完后，再加上去流向下一次循环。

**高维空间中的逻辑假设 (The Michael Jordan Example)**

- **概念深度解析**：为了探究事实是如何被储存的，视频提出了一个极其精妙的假设：在 12288 维的嵌入空间中，存在特定的“方向”代表特定的概念。 假设存在三个方向：
	- 方向 A：代表名字 `Michael`
	- 方向 B：代表姓氏 `Jordan`
	- 方向 C：代表运动 `basketball` 当一个上下文向量在经历了 Attention 的洗礼后，如果它在方向 A 和 B 上的投影（点积）都是 1，说明这个单一向量已经完美融合了“迈克尔·乔丹”这个人的复合概念。

- 大模型在做预训练（Pre-training）时，本质上就是通过海量的数据，疯狂地把全世界数以亿计的这种 `if-then` 关联逻辑，物理烧录到 MLP 庞大的矩阵参数中。

**升维投影 (Up Projection) —— 同时提出五万个“问题”**

- **概念深度解析**：当一个已经吸饱了上下文的词向量 $E$（12288 维）进入 MLP 块后，它迎来的第一步，就是乘以一个极其庞大的权重矩阵 $W_{up}$。

	这个矩阵有多大？在 GPT-3 中，它的行数是输入维度的 **4 倍**，也就是接近 **50000 行**！

- **物理意义与点积**：你可以把这个 $W_{up}$ 矩阵的**每一行**，都看作是一个独立的“特征探测器”或“问题”。

	当向量 $E$ 与第一行算点积时，就相当于在问：“你身上有『Michael + Jordan』的特征吗？”

	当它与第二行算点积时，可能在问：“你是一首诗吗？”

	以此类推，一次矩阵乘法，就是计算机在绝对并行的状态下，对这个词向量同时发起了 50000 个极其刁钻的提问。

- **偏置项 (Bias)**：算完点积后，还会加上一个偏置向量 $b_{up}$。如果偏置是负数（比如 -1），它就相当于提高了触发的门槛：只有当点积得分极高时，最终的结果才能是正数。

 **核心非线性截断：ReLU 激活函数 (相当于逻辑与门)**

- **概念深度解析**：仅仅做矩阵乘法有一个致命缺陷——它是纯线性的。如果不对分数做处理，输入 `Michael Phelps`（菲尔普斯）或者 `Alexis Jordan`，它们在这个特征探测器上可能也会得到一个正的半吊子分数，这就导致了知识的混淆。 我们需要一个“非黑即白”的严苛裁判！这里引出了深度学习中最经典的激活函数：**ReLU (Rectified Linear Unit)**。 它的逻辑极其粗暴：所有负数全部归零，所有正数原样保留。

- **案例/辅助理解 (C++ 底层思维)**： 结合你的 C++ 功底，MLP 内部这几万个神经元的激活，本质上就是一个带条件过滤的遍历：

	C++

	```CPP
	std::vector<float> neurons(50000);
	for (int i = 0; i < 50000; i++) {
	    // 1. Up Projection: 计算第 i 个特征的匹配得分，并减去阈值 (Bias)
	    float score = dot_product(W_up[i], E) + b_up[i];
	
	    // 2. ReLU 激活: 极其严苛的非线性过滤
	    // 负分直接干掉 (归零)，正分保留。这在逻辑上等同于一个 AND 门！
	    neurons[i] = std::max(0.0f, score); 
	}
	```

	只有那些在这步 `std::max` 中存活下来、值大于 0 的元素，才被称为**“被激活的神经元” (Active Neurons)**。当我们在文献里看到“Transformer 的神经元亮了”时，指的就是这个经过 ReLU 过滤后的 50000 维巨大数组里的非零值！

**降维投影 (Down Projection) —— 注入事实特征**

- **概念深度解析**：在经历了 50,000 个神经元的过滤后，我们得到了一张“激活名单”（哪些神经元亮了）。现在，我们要进行反向操作：**降维投影**。

	我们要用另一个巨大的权重矩阵 $W_{down}$（大小为 $12288 \times 50000$）来处理这个巨大的数组。

- **物理意义与写入**：如果说 $W_{up}$ 是“看行”来提问，那么 $W_{down}$ 就是**“看列”来回答**。

	矩阵的每一列其实都是一个 12288 维的向量，代表了一个特定的“语义修正包”。

	- 如果刚才探测“迈克尔·乔丹”的那个神经元亮了，那么 $W_{down}$ 中对应的那一列（恰好是“篮球”特征的方向）就会被提取出来。
	- 如果多个神经元同时亮了，模型就会把这些列对应的向量**相加**。

- **结果**：这一步产生了一个 12288 维的增量向量 $\Delta E$，它承载了模型从参数中提取出的“知识补丁”。

**残差连接 (Residual Connection)**

- **概念深度解析**：最后一步至关重要，我们并不直接用 $\Delta E$ 替换原来的词向量。

	相反，我们将这个增量 $\Delta E$ **加回到**原始流动的词向量 $E_{old}$ 上：

	$$E_{new} = E_{old} + \Delta E$$

	这就是我们在 Transformer 中反复见到的**残差连接**。

- **案例/辅助理解 (C++ 指针/缓冲区思维)**：

	想象你在维护一个名为 `ContextStream` 的全局缓冲区。数据流过 MLP 块时，MLP 并不是把原数据删了重写，而是做了一次类似“差异更新”的操作：

	C++

	```
	// 原始向量 E 就像是一个持续演化的 Buffer
	Vector delta_E = mlp_block.process(E_old); 
	
	// 残差连接：不改变 Buffer 的基调，只是往里面累加新的事实知识
	E_new = E_old + delta_E; 
	```

	这样做的好处是：即使 MLP 没提取出什么有用的事实（$\Delta E \approx 0$），原始信息（句子里的单词本身）也能毫发无伤地流向下一层，而不会在复杂的矩阵运算中丢失。

**算力巨兽 —— 真正的大头 (Parameter Count)**

- **概念深度解析**：我们先来结个账。在单层 MLP 块中，升维矩阵 $W_{up}$（约 6 亿参数）加上降维矩阵 $W_{down}$（约 6 亿参数），这一层就是 12 亿参数！

	而 GPT-3 把这个 MLP 块整整堆叠了 **96 层**。

	$12 \text{ 亿} \times 96 \approx 1160 \text{ 亿}$ 个参数。这就是为什么我们说 MLP 占据了整个大模型 1750 亿参数的**三分之二**。Attention 负责逻辑和流转，而 MLP 才是真正负责强行记忆事实的“硬盘”。

**终极前沿：叠加态 (Superposition)**

- **概念深度解析**：这是颠覆直觉的理论核心！研究发现，大模型内部极少出现“一个神经元负责一个干净概念”的情况。

	相反，它采用了**叠加态 (Superposition)**。意思是：模型并不是用 1 个专属神经元来表示“乔丹”，而是用**成百上千个神经元的微弱组合**来共同表示这个概念。

- **高维几何的魔法 (Johnson-Lindenstrauss 引理)**：

	- **严格正交的局限**：在数学中，如果你要在 $N$ 维空间里找彼此绝对垂直（$90^\circ$）的向量（互不干扰），你最多只能找到 $N$ 个。
	- **放宽条件的奇迹**：但如果你允许一点点“噪音”，比如允许向量之间的夹角在 $89^\circ$ 到 $91^\circ$ 之间。在二维或三维空间，这挤不出多少新空间。但在 **12288 维**这种极高维空间里，这种“几乎垂直”的向量数量会发生**指数级爆炸 (Exponential Growth)**！

- **物理意义**：这意味着，GPT-3 的 50,000 个神经元，不需要只存 50,000 个事实。通过允许极微小的特征重叠（噪音），它可以在这 50,000 个维度里，塞下**数以百万计、千万计**的独立事实！这就是大模型为什么“越大越强”（Scaling Laws）的底层数学原因：维度稍微增加一点，容量是呈指数级狂飙的。

**分布式表征的“黑盒”困境**

- **案例/辅助理解 (C++ 底层思维)**：

	用你的专业知识来打个比方。这就像是哈希表与 **布隆过滤器 (Bloom Filter)** 的区别。

	如果严格要求 $O(1)$ 无哈希冲突（完全正交），5 万个槽位只能存 5 万个对象。但大模型实际上在底层进化出了类似布隆过滤器的机制：每个事实用多个 bit（神经元）的特定组合来表示。虽然会有微小的特征叠加冲突（夹角不是完美的 $90^\circ$），但在超高维空间里，这种误判率被压得极低，从而实现了不可思议的极致内存压缩。

# Large Language Models explained briefly

**角色扮演与文本接龙 (The Chatbot Illusion)**

- **概念深度解析**：我们总觉得 ChatGPT 是在“思考”并“回答”我们的问题。但实际上，它的底层永远只做一件事：**根据前文，预测下一个词的概率分布**。 当我们和它聊天时，系统其实在后台悄悄拼接了一段类似电影剧本的**系统提示词 (System Prompt)**：“*现在有一个极其聪明、乐于助人的 AI 助手，人类问了它一个问题：[你的输入]，AI 的回答是：*”。模型只是顺着这个“剧本”，极其自然地把后面的台词给接力预测了出来。
- **随机性 (Temperature)**：模型本身是绝对确定性的（Deterministic 纯数学函数）。但每次回答不一样，是因为我们在它的概率输出结果中进行了**随机采样 (Random Sampling)**，允许它偶尔挑选概率稍低的词，从而产生流畅且多样的自然语言。

**核心阶段一：预训练 (Pre-training) 的暴力美学**

- **概念深度解析**：这是大模型训练的“第一重天”。模型初始的几千亿个参数完全是随机的垃圾值（输出全是乱码）。 工程师把整个互联网的文本（维基百科、Reddit、Github 代码库等）喂给它。训练方式极其简单粗暴：遮住一句话的最后一个词，让它猜；猜错了，就用我们之前学的**反向传播 (Backpropagation)** 稍微微调一下那几千亿个参数。 *算力恐怖*：如果让人类每秒进行 10 亿次加法和乘法，跑完现代大模型的预训练，需要超过 **1 亿年**的时间。这就是为什么深度学习极其依赖 GPU 并行计算。

**核心阶段二：人类反馈强化学习 (RLHF)**

- **概念深度解析**：这是现代 AI 史上的神来之笔，也是后续你在 CS336 中必须要深入研究的 Alignment（对齐）技术。    仅仅经过预训练的模型，只是一个“无情的续写机器”。如果你输入“我怎么才能制造炸弹”，它可能会顺着维基百科的语调极其专业地续写出步骤。 为了把它变成一个**“安全、听话、有帮助”的助手**，我们必须进行第二阶段的 RLHF 训练。
- **工作机制**：让人类测试员给模型的多个回答打分（比如：回答 A 很有帮助且安全，打高分；回答 B 是有毒的废话，打低分）。模型根据这些人类的反馈评分，再次更新自己的参数，**迎合人类的偏好**。
